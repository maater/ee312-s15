Program Submission Instructions

Fall 2014

We will not accept email submissions. If you email a TA or Professor Suleman your Programming Assignment, the TA/Professor Suleman will delete your email and ignore your submission. You will receive a zero on the Programming Assignment. Learn how to submit your program early so this does not happen to you.

To submit your programs for this class, you will be using a version control tool called Git. Git is a system that allows you to submit your programs to a repository where the instructors will be able to view and grade your submission.

There are four steps outlined below: installing Git, setting up your submission directory, updating your submission directory, and submitting your program. Installing Git only needs to be done once. Setting up your submission directory only needs to be done once. Both updating your submission directory and submitting your program need to be done for every programming assignment you submit.

If you are using OS X, Linux, or any other operating system, we recommend using Windows for the lab assignments. The LC-3b simulator works best on Windows, and using the Windows version will allow you to focus your work on the labs themselves. There are Windows machines available in the ECE labs, and instructions on how to use them can be found here. All of the procedures shown below should work identically on the ECE lab machines.

Finally, if you encounter issues during submission, you may attempt the following three steps in sequence:

  1. Backup your files/programs. You can do this by copying your files/programs into another directory, which you should not delete.

  2. Delete your entire submission directory

  3. Repeat the steps below beginning with setting up your submission directory.

Remember: You are responsible for the security of your files. Do NOT allow anyone to access your files! If someone cheats by accessing your files because they were not properly secured, both parties will be found guilty of cheating and suffer the same consequences.

Having problems submitting? Contact a TA in person during office hours.

Installing Git (Windows)

  1. Navigate to http://git-scm.com/download/win. The download for Git for Windows should begin automatically.
  2. Run the installer file you downloaded in step 1.
  3. Accept all the default settings by clicking Next until the installation is complete.
  4. If Windows asks for permission to install Git, click Yes.
  5. If Git asks you to restart your computer, click Yes (this will immediately restart your computer. Save your work beforehand).

Setting Up Your Submission Directory

For this part, you will need to have created a Github account and installed Git for Windows.
  1. Create a directory on the machine you are using where you can work on and submit EE306 labs. For example, you may call it "EE306_Labs".
  2. Enter this diretory and right click on empty white space. Click on the option labeled "Git Bash".



  3. A window in which you can type commands should popup. Type the following command and press Enter: 
    git config --global user.name "Your Name"
  4. Type the following command and press Enter: 
    git config --global push.default simple
  5. Type the following command and press Enter: 
    git config --global user.email "your_email@utexas.edu"



  6. Now that you have set up the configuration options, you must get a copy of your repository so that you can make changes and upload them for the instructors to see. You can do this by typing in the following command and pressing Enter: 
    git clone https://github.com/ee306-f14/<EID>.git
    Where <EID> should be replaced with your EID. For example: 
    git clone https://github.com/ee306-f14/xyz123.git
    If you are prompted to enter your Github username and password, please enter it. Note that your password will not be shown as you type it. You should get a reuslt similar to what you see below.



  7. You should see a new directory appear in the directory you created in step 1. This directory will be your submission directory. Inside the submission directory, you will see a series of files pertaining to the labs. You will modify these existing files with the requirements for the lab and then upload them for submission.

Note: If you see folders called ".git", do not modify/delete them.

Updating Your Submission Directory

For this part, you will need to have created a Github account and set up your submission directory.

You MUST perform this procedure before beginning any of the lab assignments.

  1. Enter the submission directory created in "Setting Up Your Submission Directory" (e.g., EE306_Labs\EID). Right click on empty white space. Click on the option labeled "Git Bash".



  2. Type the following command and press Enter: 
    git pull
    If you are prompted to enter your Github username and password, please enter it. Note that your password will not be shown as you type it. The output may not necessarily match what you see below. If the output differs from what you see below, check the submission directory and you should see new files added.

Submitting Your Program

Once you have completed the the lab, you are ready to submit your files. The individual lab assignments will specify which files you should modify for that lab. Only modify files that are specified in the current lab assignment you are working with.

Note: You can resubmit your program as many times as you want up until the deadline. We will ignore any submissions you make beyond the deadline, so be sure to finish your lab on time.

  1. Enter the submission directory created in "Setting Up Your Submission Directory" (e.g., EE306_Labs\EID). Right click on empty white space. Click on the option labeled "Git Bash".



  2. Type the following command and press Enter: 
    git commit -a -m "message"
    Please type in a meaningful message inside the quotation marks. For example, if you have completed the lab assignment, say something along the lines of "Completed Lab 0".

    This command saves all the changes you have made to the files inside the repository. This will only saves the changes on your local machine, meaning the instructors will not have access to your lab solution until you complete step 3.



  3. Once you have saved the changes, you must upload it to Github so that the instructors can access it. To upload your code, type the following command and press Enter: 
    git push
    If you are prompted to enter your Github username and password, please enter it. Note that your password will not be shown as you type it. You should get a result similar to what you see below.



    Congratulations! You've submitted your program.

Verifying Your Submission

Once you have submitted your solution, you can check to make sure the instructors have access to it.

  1. In your browser, navigate to https://github.com/ee306-f14/<EID>/commits/master where <EID> is replaced with your EID.
  2. You should see your latest submission listed on the webpage.


  3. Try refreshing your browser if it appears that the file on the website is not updating.

Note: If you are familiar with command-line based version control tools, feel free to use these at your own risk. We are not responsible if you fail to correctly submit your program with these tools.
